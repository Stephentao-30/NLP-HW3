Uncertainty Sets for Image Classifiers using Conformal Prediction

Summary
This blog post will teach you an algorithm which quantifies the uncertainty of   any classifier on any dataset in finite samples for free .
The algorithm, called  RAPS , modifies the classifier to output a predictive set containing the true label with a user-specified probability, such as 90%.
This  coverage level  is formally guaranteed even when the dataset has a finite number of samples.
Empirically, the sets are also relatively small on Imagenet with a ResNet-152, which suggests they will be effective to quantify uncertainty in high-risk settings such as medical diagnostics.
Our  GitHub implementation  allows you to use the method on any classifier, and you can try it out in Colab without downloading anything.
tl; dr
You are given a small holdout set of labelled images $(x_i,y_i)_{i=1}^n$ where $n$ is e.g. $1000$, a pretrained classifier which takes in $x$ and outputs the score of class $y$ as $\hat{\pi}_y(x)$, and a new image $x_{n+1}$.
Running the following algorithm will return  a set of labels which contains the true label $y_{n+1}$ with high probability $1-\alpha$ , no matter the model or dataset, in finite samples.
\begin{algorithm}
\caption{ RAPS }
\begin{algorithmic}
\PROCEDURE{RAPS}{the calibration dataset, the model, the new image}
\STATE \textbf{calibrate:} perform Platt scaling on the model using the calibration set.
\STATE \textbf{get conformal scores:} For each image in the training set, define $E_j=\sum_{i=1}^{k'}(\hat{\pi}_{(i)}(x_j)+\lambda \mathbb{1}[i > k_{reg}])$ where $k'$ is the model's ranking of the true class $y_j$, and $\hat{\pi}_{(i)}(x_j)$ is the $i^{th}$ largest score for the $j^{th}$ image.
\STATE \textbf{find the threshold:} assign $\hat{\tau}_{ccal}$ to the $1-\alpha$ quantile of the $E_j$.
\STATE \textbf{predict:} Output the $k^*$ highest-score classes, where $\sum_{i=1}^{k^*}(\hat{\pi}_(i)(x_{n+1})+\lambda \mathbb{1}[j > k_{reg}]) \geq \hat{\tau}_{ccal}$.
\ENDPROCEDURE
\end{algorithmic}
\end{algorithm}
Intuitively, a high $\lambda$ discourages sets larger than $k_{reg}$.
For example, if $\lambda=1$ and we choose $k_{reg}$ properly, no sets larger than $k_{reg}$ will be produced.
We include automated choices of these parameters that optimize either to minimize size or to most faithfully articulate the model's uncertainty on a particular example; see the paper for details on this.
The  RAPS  method easily wraps around any classifier with  our GitHub .
After installing our dependencies, you can conformalize your pretrained  model  and output a predictive set with, e.g.,
model = ConformalModel(model, calib_loader, alpha=0.1, lamda_criterion='size')
logits, raps_sets = model(x)

Motivation
Imagine you are a doctor making a high-stakes medical decision based on diagnostic information from a computer-vision classifier.
What would you want the classifier to output in order to make the best decision?
As a doctor, your primary objective is to ensure the health of your patient, which requires ruling in or ruling out harmful diagnoses.
In other words, even if the most likely diagnosis is a stomach ache, it is equally or more important to rule out stomach cancer.
Therefore, you would want the classifier to give you actionable uncertainty quantification, such as a set of predictions that provably covers the true diagnosis with a high probability (e.g. 90%).
This is called a  prediction set  (see Figure 2), and it is analogous to a confidence interval for black box classifiers like neural networks.
The  RAPS  paper  describes a new way to construct prediction sets from any pre-trained image classifier that have exactly the right confidence and small size.
RAPS  is as simple as Platt scaling and computationally trivial.
Objectives
Figure 2. Prediction set examples . The sets are smaller for the easy squirrel than for the hard squirrel. This is good, because the set size faithfully articulates the uncertainty of the model! These sets were produced by  RAPS  with a ResNet-152 at the 95% level. Underneath each class is the temperature-scaled softmax score of that class. Notice that they do not sum to 95%. As a scientist looking at these sets, you might flag all predictions that have large sets to take a closer look, since the model's top-1 answer could be unreliable.
We are given the following:
a  calibration dataset , $\{(x_i,y_i)\}_{i=1}^n \overset{\rm i.i.d.}{\sim} \mathbb{P}$ where $x_i \in \mathbb{R}^d$ and $y_i \in \mathcal{Y}=\{1,...,K\}$. For example, $(x_i,y_i)$ could be an image-label pair;
a  pretrained model , $\hat{\pi}_y(x)$, estimating the true conditional distribution $\pi_y(x)=\mathbb{P}\big[Y=y|X=x\big]$;
and a  new example , $x_{n+1}$.
We would like to predict a  set  $\mathcal{C}(x_{n+1})\subseteq \mathcal{Y}$ that  provides coverage : it contains the true class $y_{n+1}$ with high probability.
A function $\mathcal{C}:\mathbb{R}^d\to\mathcal{Y}$ has  conservative $1-\alpha$-coverage  if $\mathbb{P}[y_i \in \mathcal{C}(x_i)] \geq 1-\alpha$.  Exact coverage  means equality holds.
Subject to the coverage constraint, we would like the sets to be small on average.
Finally, we want them to be  adaptive : the sets should be smaller for easy examples than for hard ones.
RAPS  learns $\mathcal{C}$ from the dataset to have exact coverage and output small and adaptive sets.
Practically, the method works on large-scale, realistic datasets like Imagenet (see Figure 3).
Figure 3. Coverage and average set size on Imagenet for prediction sets from three methods . The  RAPS  procedure achieves exact coverage and has 5-10x smaller sets than competing methods. All methods use a ResNet-152 as the base classifier, and results are reported for 100 random splits of Imagenet-Val, each of size 20K. See the paper for details.
The Naive Algorithm
One naive approach is to assemble the set by including classes from highest to lowest probability (e.g. after Platt scaling and a softmax function) until their sum just exceeds $1-\alpha$.
This procedure is summarized in the algorithm below.
\begin{algorithm}
\caption{naive}
\begin{algorithmic}
\PROCEDURE{naive}{the calibration dataset, the model, the new image}
\STATE Perform Platt scaling on the model using the calibration dataset.
\STATE Compute the softmax scores by running the model on the new image.
\STATE Chosose classes from highest to lowest score until their total score just exceeds $1-\alpha$.
\ENDPROCEDURE
\end{algorithmic}
\end{algorithm}
There are two problems with the naive method: first, the probabilities output by CNNs are known to be incorrect, so the sets do not achieve coverage.
Second, for examples where the model is not confident, the naive strategy must select many classes before it reaches the desired confidence level, leading to a large set size.
Temperature scaling cannot solve this problem, since it only calibrates the score of the top class, and it is intractable to calibrate the rest.
Perhaps surprisingly, even if all scores were perfectly calibrated, naive still would not achieve coverage.
An illustration of the naive method . Since the softmax scores are not the true probabilities, the pink threshold does not provide coverage.
Regularized adaptive prediction sets
Conformal prediction is a simple idea at heart.
Instead of taking the softmax scores at face value, we learn a new threshold using the calibration dataset.
For example, with $\alpha=10%$, if choosing sets that contain 93% estimated probability achieve 90% coverage on the holdout set, we use the 93% cutoff instead.
We refer to this algorithm, introduced  here  by Romano et al., as  Adaptive Prediction Sets  ( APS ).
APS  is a special case of  RAPS  with $\lambda=0$ (see Algorithm 1).
As we will see in a moment,  APS  provably achieves exact coverage.
However,  APS  has a problem in practice: the average set size is quite large.
Deep learning classifiers suffer from a  permutation problem : the scores of the less confident classes (e.g. classes 10 through 1000) are not reliable probability estimates.
The ordering of these classes is primarily determined by noise, so  APS  has to take very large sets for some difficult images.
The  RAPS  procedure (detailed in Algorithm 1) discourages these overly large sets with regularization.
Every class beyond the $k_{reg}$ most likely classes is subject to a constant regularization penalty $\lambda$ which discourages its inclusion in the predictive set.
Still,  RAPS  achieves the same coverage, as we show below:
(APS and RAPS achieve exact coverage)
Assume the dataset $(X_i,Y_i,U_i)_{i=1,...,n}$ and $(X_{n+1},Y_{n+1})$ are i.i.d. and let the (randomized) prediction set function $\mathcal{C}$ be chosen with  APS  or  RAPS . Then we have the following coverage guarantee:
\begin{equation}
1-\alpha \leq \mathbb{P}\big[Y_{n+1} \in \mathcal{C}(X_{n+1})\big] \leq 1-\alpha+\frac{1}{n+1}.
\end{equation}
A more formal statement of the theorem and its proof reside in Theorem 1 and Appendix A of the  RAPS  paper.
An illustration of the RAPS method . The red cutoff, $\hat{\tau}_{ccal}$, achieves exact coverage. This example shows RAPS with $\lambda=0.1$ and $k_{reg}=2$ (i.e., a penalty of $0.1$ is applied to all but the top 2 scores).

This method enables a researcher to take any base classifier and return predictive sets guaranteed to achieve a pre-specified error level, such as 90%, while retaining small average size.
It is simple to deploy, so it is an attractive, automatic way to quantify the uncertainty of image classifiers, an essential task in settings like medical diagnostics, self-driving vehicles, and flagging dangerous internet content.
Predictive sets in computer vision have many further uses, since they systematically identify hard test-time examples.
In active learning, finding such examples reduces the number of points that must be labelled.
In a different direction, one can improve efficiency of a classifier by using a cheap classifier to output a prediction set first, and an expensive one only when the cheap classifier outputs a large set (a  cascade ).
Finally, one can use predictive sets during model development to identify failure cases and outliers.
This helps probe the model's weaknesses, suggesting strategies for improving its performance.
